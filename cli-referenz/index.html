Port-Konflikt (18789 belegt)
Ein anderer Prozess belegt den Gateway-Port.
lsof -i :18789 # Prozess beenden: kill -9 <PID>
Context voll (Token-Limit erreicht)
Kontext zusammenfassen oder neue Session starten.
/compact # Oder komplett neu: /new
Channel down (Kanal offline)
Kanal-Status pruefen und neu verbinden.
openclaw channels status --probe # Telegram neu verbinden: openclaw channels login telegram
WhatsApp-Verbindung reisst ab
QR-Code neu scannen. WhatsApp Web muss auf dem Handy aktiv bleiben.
openclaw channels login whatsapp # QR-Code im Terminal scannen
Config invalid (JSON-Syntaxfehler)
Config-Datei validieren. JSON5 erlaubt Kommentare + Trailing Commas.
openclaw config validate
Auth/Pairing schlaegt fehl
Ab v2026.3.7+: Wenn sowohl Token als auch Passwort gesetzt sind, MUSS gateway.auth.mode explizit auf 'token' oder 'password' stehen.
// In openclaw.json: "gateway": { "auth": { "mode": "token", // EXPLIZIT setzen! "token": "dein-token" } }
Restart-Loop (Gateway startet endlos neu)
Oft ein Config-Problem. Update auf mindestens v2026.2.22+.
docker logs openclaw --tail 50 openclaw config validate openclaw update --channel stable
Disk full (Speicherplatz voll)
Alte Sessions und Docker-Artefakte aufraeumen.
openclaw sessions cleanup docker system prune -f df -h
Container laeuft nicht
Logs pruefen und Container neu starten.
docker logs openclaw --tail 100 docker restart openclaw # Oder komplett: docker compose down && docker compose up -d
Tailscale verbindet nicht
Tailscale-Daemon neu starten und Status pruefen.
sudo tailscale status sudo systemctl restart tailscaled sudo tailscale up
