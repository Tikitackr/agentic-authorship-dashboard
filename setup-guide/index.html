Kapitel 2: Installation & Setup
7 Schritte

Waehle deinen Installationsweg. Das Buch arbeitet mit Hostinger (VPS), aber du kannst auch lokal starten.

Lokal installieren

npm auf deinem Mac/Linux/Windows (WSL2). Schneller Start, kein Server noetig.

Buch-Empfehlung

VPS mit Hostinger

24/7 erreichbar, vom Handy steuerbar. ~5 EUR/Monat. Der Weg den das Buch geht.

Schritt 1 machst du im Hostinger-Panel (Browser). Ab Schritt 2 per SSH auf dem VPS.

1. OpenClaw im Hostinger Docker Manager deployen

Hostinger Panel → Docker Manager → Katalog → OpenClaw → „Bereitstellen". Trag deinen API-Key und Gateway Token ein. Merk dir den Port (z.B. 52127).

Wichtig: Dein Agent laeuft jetzt auf dem Server, aber du kannst ihn noch nicht oeffnen. Hostinger liefert nur HTTP, dein Browser braucht aber eine sichere HTTPS-Verbindung. In den naechsten Schritten richten wir das mit Tailscale ein (kostenlos, 2 Minuten).
2. Per SSH mit dem VPS verbinden
ssh root@${SERVER_IP}
3. Tailscale installieren
curl -fsSL https://tailscale.com/install.sh | sh
4. Tailscale starten & authentifizieren
sudo tailscale up

Kopiere den angezeigten Link, oeffne ihn im Browser, melde dich bei Tailscale an und klicke „Connect".

5. Tailscale Serve (HTTPS fuer deinen Server)
sudo tailscale serve --bg http://localhost:${PORT}

Notiere dir die angezeigte HTTPS-Adresse (z.B. https://srv1192770.tail755c12.ts.net/) und trage sie unter „Meine Daten" ein.

6. Tailscale auf Mac/Handy installieren

App Store → „Tailscale" → installieren → mit demselben Account anmelden. Auf dem Handy genauso.

7. OpenClaw im Browser öffnen

Klick auf den Link oder kopiere die Adresse in deinen Browser:

https://${HOSTNAME}:${PORT}/#token=${GATEWAY_TOKEN}
Gateway Token bereithalten! Du landest auf einer Login-Seite. Kopiere deinen Gateway Token vorher in den Zwischenspeicher (Auge-Symbol bei „Meine Daten" → markieren → kopieren). Den Token auf der Login-Seite einfügen und auf „Login" klicken.

Du siehst das OpenClaw-Dashboard? Glückwunsch, dein Agent läuft!

Lesezeichen setzen! Diese Adresse ist ab jetzt dein täglicher Zugang zu OpenClaw. Setz ein Lesezeichen im Browser und auf dem Handy.
1. OpenClaw installieren
npm install -g openclaw@latest
2. Einrichtungsassistent
openclaw onboard --install-daemon
3. Gateway starten
openclaw gateway --port 18789
4. Testen
openclaw status
5. Im Browser oeffnen
http://127.0.0.1:18789
Kapitel 3: Docker-Befehle
6 Befehle

Wichtige Docker-Befehle zum Verwalten deiner OpenClaw-Container.

1. Container-Status
docker ps
2. Logs anzeigen (Live)
docker logs ${CONTAINER_NAME} -f
3. Container neustarten
docker restart ${CONTAINER_NAME}
4. In Container einsteigen (Bash)
docker exec -it ${CONTAINER_NAME} bash
5. Docker Compose starten
docker compose up -d
6. Docker Compose stoppen
docker compose down
Kapitel 5: Tailscale & Multi-Channel
6 Schritte

Verbinde deinen Server über Tailscale und konfiguriere Telegram-Integration.

1. Tailscale installieren
curl -fsSL https://tailscale.com/install.sh | sh
2. Tailscale starten
tailscale up --hostname=${HOSTNAME}
3. Tailscale Status prüfen
tailscale status
4. Tailscale Serve aktivieren (HTTPS fuer Dashboard)

Macht deinen Sync-Server ueber HTTPS erreichbar. Den Hostnamen (z.B. srv1234.tail5678.ts.net) traegst du unter "Meine Daten" ein.

tailscale serve --bg 3456
5. Telegram-Bot konfigurieren (optional)
docker exec ${CONTAINER_NAME} openclaw config set telegram.botToken "${TELEGRAM_TOKEN}"
6. Telegram erlaubte User
docker exec ${CONTAINER_NAME} openclaw config set telegram.allowFrom '["${TELEGRAM_ID}"]'
Kapitel 6: Security
4 Schritte

Sichere deine OpenClaw-Installation mit Firewall und Authentifizierung.

1. Firewall einrichten
ufw allow 22/tcp && ufw allow 41641/udp && ufw enable
2. Gateway-Token setzen
docker exec ${CONTAINER_NAME} openclaw config set gateway.token "${GATEWAY_TOKEN}"
3. Bind-Modus prüfen
docker exec ${CONTAINER_NAME} openclaw config get gateway.bind
4. CORS-Whitelist
docker exec ${CONTAINER_NAME} openclaw config set gateway.cors '["https://openclaw-buch.de"]'
Kapitel 9: Workspace-Dateien
2 Befehle

Die Workspace-Dateien (SOUL.md, AGENTS.md, BOOT.md) generierst du am besten mit dem Workspace Wizard.

1. Workspace synchronisieren (rsync)
rsync -avz ~/mein-buch/.openclaw/workspace/ root@${TAILSCALE_IP}:/docker/${CONTAINER_NAME}/data/.openclaw/workspace/
2. Permissions reparieren
ssh root@${TAILSCALE_IP} 'chown -R 1000:1000 /docker/${CONTAINER_NAME}/data/.openclaw/workspace/'
