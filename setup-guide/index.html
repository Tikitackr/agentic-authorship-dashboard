Kapitel 2: Server-Setup (Installation)
6 Schritte

Verbinde dich mit deinem Server und richte OpenClaw ein.

1. SSH verbinden
ssh root@${SERVER_IP}
2. System aktualisieren
apt update && apt upgrade -y
3. Docker installieren
curl -fsSL https://get.docker.com | sh
4. OpenClaw starten
docker run -d --name ${CONTAINER_NAME} --restart unless-stopped -p 8443:8443 -v /docker/${CONTAINER_NAME}/data:/data ghcr.io/openclaw/openclaw:latest
5. Gateway prüfen
docker logs ${CONTAINER_NAME} --tail 50
6. API-Key setzen
docker exec ${CONTAINER_NAME} openclaw config set anthropic.apiKey "${API_KEY}"
Kapitel 3: Docker-Befehle
6 Befehle

Wichtige Docker-Befehle zum Verwalten deiner OpenClaw-Container.

1. Container-Status
docker ps
2. Logs anzeigen (Live)
docker logs ${CONTAINER_NAME} -f
3. Container neustarten
docker restart ${CONTAINER_NAME}
4. In Container einsteigen (Bash)
docker exec -it ${CONTAINER_NAME} bash
5. Docker Compose starten
docker compose up -d
6. Docker Compose stoppen
docker compose down
Kapitel 5: Tailscale & Multi-Channel
6 Schritte

Verbinde deinen Server über Tailscale und konfiguriere Telegram-Integration.

1. Tailscale installieren
curl -fsSL https://tailscale.com/install.sh | sh
2. Tailscale starten
tailscale up --hostname=${HOSTNAME}
3. Tailscale Status prüfen
tailscale status
4. Tailscale Serve aktivieren (HTTPS fuer Dashboard)

Macht deinen Sync-Server ueber HTTPS erreichbar. Den Hostnamen (z.B. srv1234.tail5678.ts.net) traegst du unter "Meine Daten" ein.

tailscale serve --bg 3456
5. Telegram-Bot konfigurieren (optional)
docker exec ${CONTAINER_NAME} openclaw config set telegram.botToken "${TELEGRAM_TOKEN}"
6. Telegram erlaubte User
docker exec ${CONTAINER_NAME} openclaw config set telegram.allowFrom '["${TELEGRAM_ID}"]'
Kapitel 6: Security
4 Schritte

Sichere deine OpenClaw-Installation mit Firewall und Authentifizierung.

1. Firewall einrichten
ufw allow 22/tcp && ufw allow 41641/udp && ufw enable
2. Gateway-Token setzen
docker exec ${CONTAINER_NAME} openclaw config set gateway.token "${GATEWAY_TOKEN}"
3. Bind-Modus prüfen
docker exec ${CONTAINER_NAME} openclaw config get gateway.bind
4. CORS-Whitelist
docker exec ${CONTAINER_NAME} openclaw config set gateway.cors '["https://openclaw-buch.de"]'
Kapitel 9: Workspace-Dateien
2 Befehle

Die Workspace-Dateien (SOUL.md, AGENTS.md, BOOT.md) generierst du am besten mit dem Workspace Wizard.

1. Workspace synchronisieren (rsync)
rsync -avz ~/mein-buch/.openclaw/workspace/ root@${TAILSCALE_IP}:/docker/${CONTAINER_NAME}/data/.openclaw/workspace/
2. Permissions reparieren
ssh root@${TAILSCALE_IP} 'chown -R 1000:1000 /docker/${CONTAINER_NAME}/data/.openclaw/workspace/'
