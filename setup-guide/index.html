Kapitel 2: Installation & Setup
7 Schritte

Waehle deinen Installationsweg. Das Buch arbeitet mit Hostinger (VPS), aber du kannst auch lokal starten.

Lokal installieren

npm auf deinem Mac/Linux/Windows (WSL2). Schneller Start, kein Server noetig.

Buch-Empfehlung

VPS mit Hostinger

24/7 erreichbar, vom Handy steuerbar. ~5 EUR/Monat. Der Weg den das Buch geht.

Schritt 1 machst du im Hostinger-Panel (Browser). Ab Schritt 2 per SSH auf dem VPS.

1. OpenClaw im Hostinger Docker Manager deployen

Hostinger Panel → Docker Manager → Katalog → OpenClaw → „Bereitstellen". Trag deinen API-Key und Gateway Token ein. Merk dir den Port (z.B. 52127).

Wichtig: Dein Agent läuft jetzt auf dem Server, aber du kannst ihn noch nicht öffnen. Hostinger liefert nur HTTP, dein Browser braucht aber eine sichere HTTPS-Verbindung. In den nächsten Schritten richten wir das mit Tailscale ein (kostenlos, 2 Minuten).
2. Per SSH mit dem VPS verbinden
ssh root@${SERVER_IP}
3. Tailscale installieren
curl -fsSL https://tailscale.com/install.sh | sh
4. Tailscale starten & authentifizieren
sudo tailscale up

Kopiere den angezeigten Link, öffne ihn im Browser, melde dich bei Tailscale an und klicke „Connect".

5. Tailscale Serve (HTTPS für deinen Server)
sudo tailscale serve --bg http://localhost:${PORT}

Notiere dir die angezeigte HTTPS-Adresse (z.B. https://srv1192770.tail755c12.ts.net/) und trage sie unter „Meine Daten" ein.

6. Tailscale auf Mac/Handy installieren

App Store → „Tailscale" → installieren → mit demselben Account anmelden. Auf dem Handy genauso.

7. OpenClaw im Browser öffnen

Zwei Klicks bis zu deinem Dashboard:

2 Dashboard öffnen

Auf der Login-Seite den Token einfügen und auf „Login" klicken. Danach Lesezeichen setzen!

Du siehst das OpenClaw-Dashboard? Glückwunsch, dein Agent läuft!

Lesezeichen setzen! Diese Adresse ist ab jetzt dein täglicher Zugang zu OpenClaw. Setz ein Lesezeichen im Browser und auf dem Handy.
1. OpenClaw installieren
npm install -g openclaw@latest
2. Einrichtungsassistent
openclaw onboard --install-daemon
3. Gateway starten
openclaw gateway --port 18789
4. Testen
openclaw status
5. Im Browser öffnen
http://127.0.0.1:18789
Kapitel 3: Docker-Befehle
12 Befehle
Hostinger-Nutzer?

Der Docker Manager im hPanel erledigt das meiste per Klick: Container starten, stoppen, Logs ansehen, Updates einspielen. Die folgenden 12 Terminal-Befehle brauchst du vor allem bei einer lokalen Installation oder wenn du tiefer einsteigen willst. Sie zeigen, was Docker unter der Haube macht.

1. Container-Status
docker ps
2. Logs anzeigen (Live)
docker compose logs -f
3. Container neustarten
docker compose restart
4. In Container einsteigen (Bash)
docker exec -it ${CONTAINER_NAME} bash
5. Docker Compose starten
docker compose up -d
6. Docker Compose stoppen
docker compose down
7. OpenClaw Version pruefen
docker exec -it ${CONTAINER_NAME} openclaw version
8. OpenClaw Status
docker exec -it ${CONTAINER_NAME} openclaw status
9. Diagnose (openclaw doctor)
docker exec -it ${CONTAINER_NAME} openclaw doctor
10. Auto-Reparatur
docker exec -it ${CONTAINER_NAME} openclaw doctor --fix
11. Update: Neues Image ziehen + Container neu erstellen
docker compose pull && docker compose up -d --force-recreate
12. Aufraeumen: Alte Images entfernen
docker system prune -f
Kapitel 5: Multi-Channel Setup
6 Schritte

Messaging-Kanäle richtest du per Chat mit deinem Agenten ein. Hier das Telegram-Tutorial Schritt für Schritt.

Buch-Empfehlung
Telegram einrichten (der einfache Weg)

Telegram ist der Goldstandard. Offizielles Bot-API, stabil, schnell. Alles per Chat, kein Terminal.

1. Bot-Token holen bei @BotFather

Öffne Telegram, suche @BotFather und schick ihm:

/newbot

BotFather fragt nach einem Namen (z.B. „Mein OpenClaw Agent“) und einem Username (muss auf bot enden). Dann bekommst du den Token – sieht aus wie 1234567890:ABCdef...

💡 Token vergessen? Schick /mybots an @BotFather – er zeigt dir alle deine Bots und deren Tokens.

2. Token deinem Agent geben

Schreib deinem Agent (im Gateway-UI):

Richte mir Telegram ein. Mein Token ist: ${TELEGRAM_TOKEN}

Der Agent speichert den Token und sagt dir, was als nächstes kommt.

3. Bot finden und Nachricht schicken

Suche deinen Bot in Telegram (den Username von Schritt 1) und schick ihm irgendwas, z.B. „Hallo“. Er antwortet: „Access not configured. Pairing required.“

4. Pairing bestätigen

Im Gateway siehst du eine Pairing-Anfrage mit einem Code. Sag dem Agent:

Genehmige die Kopplung.

Ab jetzt antwortet der Bot dir auf Telegram.

5. ID dauerhaft eintragen

Damit die Kopplung einen Container-Restart überlebt:

Trag meine Telegram-ID fest in die Config ein.

Der Agent liest deine ID aus dem Pairing und speichert sie dauerhaft. Fertig!

💡 User-ID vergessen? Suche @userinfobot in Telegram und schick ihm eine Nachricht – er antwortet sofort mit deiner numerischen ID.

Tipp: Trag deinen Telegram-Token und deine User-ID unter Meine Daten ein – dann hast du beides immer griffbereit und es wird im Setup-Guide automatisch eingesetzt.

6. Weitere Kanäle (gleicher Weg)

Das Prinzip ist immer gleich: Agent fragen, er führt dich durch.

Richte mir WhatsApp ein.

⚠️ Nutze eine separate Nummer! Kein offizielles API, Account-Sperre möglich.

Richte mir Discord ein.

Für Communities und Teams. Bot im Developer Portal erstellen, Agent macht den Rest.

Führe den Doktor-Check aus.

Bei Problemen. Prüft Tokens, IDs, Config und repariert automatisch.

Terminal-Befehle für Fortgeschrittene findest du im Buch unter Kapitel 5.2 / 5.3.

Kapitel 6: Security
5 Schritte

Dein Setup bringt vier Schutzschichten mit: Tailscale, Token, Sandbox, Tool-Policies. Hier prüfst du, ob alles sauber läuft.

Security-Routine
Alles per Chat, kein Terminal

Auf deinem Hostinger VPS läuft alles im Docker Manager. Die Security-Checks machst du per Chat mit deinem Agenten.

1. Bind-Config prüfen

Prüft, ob dein Gateway nur im Tailnet lauscht (und nicht öffentlich erreichbar ist):

Zeig mir die aktuelle Bind-Config.

💡 Erwartet: tailnet oder loopback. Steht dort 0.0.0.0? Sofort ändern!

2. Token rotieren

Alle paar Monate oder wenn du unsicher bist, ob dein Token irgendwo sichtbar war:

Generiere mir einen neuen Gateway-Token und trag ihn in die Config ein.

Drei Sekunden. Alter Token wertlos, neuer Token aktiv. Danach unter Meine Daten aktualisieren.

3. Doktor-Check

Prüft Config, Permissions und Dependencies. Nach jedem Update oder einmal im Monat:

Führe den Doktor-Check aus und behebe alle Probleme.
4. Security-Audit

Geht tiefer als der Doktor-Check. Prüft Bind, Auth, File-Permissions, Sandbox:

Führe einen Security-Audit durch.
5. Update prüfen

Updates enthalten Security-Fixes. Regelmäßig prüfen:

Gibt es ein OpenClaw-Update? Wenn ja, installiere es.
Bonus: Verbundene Geräte prüfen

Falls du prüfen willst, welche Geräte Zugang haben:

Zeig mir alle wartenden Pairing-Anfragen.

Hostinger-Tipp: UFW (Firewall) musst du auf Hostinger nicht manuell einrichten. Der Docker Manager regelt die Netzwerk-Isolation. Tailscale sorgt dafür, dass dein Gateway nur in deinem Tailnet erreichbar ist.

Kapitel 7: Troubleshooting
6 Schritte + Bonus

Die Command Ladder als Chat-Prompts. Von schnell bis tief, systematisch statt raten.

Troubleshooting-Routine
Der Doctor löst ~70% aller Probleme automatisch

Starte immer beim Schnell-Check und arbeite dich nach unten. Meistens bist du nach Schritt 3 fertig.

1. Schnell-Check

Läuft der Gateway? Welcher Agent ist aktiv? Wie viele Sessions sind offen?

Zeig mir den aktuellen Status.
2. Deep Status (API-Provider testen)

Prüft ob Anthropic, OpenAI und alle anderen Provider wirklich erreichbar sind:

Prüfe ob alle API-Provider erreichbar sind.

💡 Dauert ein paar Sekunden länger, weil echte Test-Requests an die APIs geschickt werden.

3. Kanal-Verbindungstest

Telegram, WhatsApp, Discord: Sind sie wirklich verbunden oder nur konfiguriert?

Teste ob alle Kanäle verbunden sind.
4. Doktor + automatische Reparatur

19 Diagnose-Schritte. Findet UND repariert die meisten Probleme:

Führe den Doktor-Check aus und behebe alle Probleme.
5. Logs zusammenfassen

Wenn der Doctor nichts findet, schau in die Logs:

Fasse die letzten Fehler aus den Logs zusammen.
6. Hängende Sessions finden

Offene Sessions blockieren Ressourcen und verlangsamen alles:

Zeig mir alle aktiven Sessions der letzten 2 Stunden.
Bonus: /debug Befehle (direkt im Chat tippen)

Zum Experimentieren, ohne die Config zu ändern. Änderungen gelten nur für die aktuelle Session.

/debug model=anthropic/claude-haiku-4

Wechselt das Modell für diese Session (z.B. um Geschwindigkeit zu testen).

/debug thinking=high

Dreht die Reasoning-Stufe hoch (genauer, aber langsamer).

Hostinger-Tipp: Auf deinem VPS läuft OpenClaw im Docker-Container. Du brauchst kein docker exec zu tippen. Alle Diagnose-Befehle laufen per Chat mit deinem Agenten.

Kapitel 9: Workspace-Dateien
2 Befehle

Die Workspace-Dateien (SOUL.md, AGENTS.md, BOOT.md) generierst du am besten mit dem Workspace Wizard.

1. Workspace synchronisieren (rsync)
rsync -avz ~/mein-buch/.openclaw/workspace/ root@${TAILSCALE_IP}:/docker/${CONTAINER_NAME}/data/.openclaw/workspace/

ssh root@${TAILSCALE_IP} 'chown -R 1000:1000 /docker/${CONTAINER_NAME}/data/.openclaw/workspace/'
